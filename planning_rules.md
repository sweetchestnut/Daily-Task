# Planning Rules

## Core Rules
- Never schedule more than 3 core tasks per day
- Prioritize tasks with high uncertainty before routine tasks
- Prefer batching similar tasks together
- Always show a draft schedule first

## Time Block Rules
- Do not schedule formal work before 09:00 by default
- Place deep work tasks into:
  - 10:00-11:30
  - 14:00-17:00
  - 19:00-21:30
- After 21:30, only allow light tasks:
  - review
  - follow-up
  - organizing
  - low-cognitive admin work

## Priority Ordering
Suggested ranking order:
1. P0 over P1 over P2
2. Higher uncertainty first
3. Earlier deadline first
4. Greater project impact first
5. Tasks that can be batched together

## Device / Context Rules
Tasks should be tagged when possible:
- deep_work
- computer_required
- mobile_ok
- offline_ok
- light_task

## Output Format
The final daily output should always include:
1. Today's Top 3
2. Time-block schedule
3. Delay suggestions
4. Progress / change reminders
